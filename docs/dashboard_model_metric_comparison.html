What the chart shows

Across all models (4B–35B) and all harnesses, the unknown seizure type miss count remains 13–15 out of 26 gold instances. Scale does not reduce this gap. The label is a meta-judgment (annotator cannot determine type), but models consistently infer a specific type from context.

Oracle failure rate

13.3% of validation documents have annotation gaps that make seizure type unscoreable even with perfect extraction. Combined with the meta-label mismatch, this explains why seizure type F1 plateaus around 0.60–0.63.

Letter Excerpt (EA00xx)

"She has been seizure free for the last 18 months. Previously she experienced focal seizures with impaired awareness, but these have not recurred since her medication was optimised."

Gold Standard

seizure_types: ["seizure free"]
epilepsy_diagnosis: "focal epilepsy"
medications: ["levetiracetam 1000mg twice daily"]

qwen3.5:9b H6 (baseline) — ERROR

seizure_types: ["focal seizure with impaired awareness"]
← Hallucination: extracts historical type as current

qwen3.5:9b H6fs (few-shot) — CORRECTED

seizure_types: ["seizure free"]
← Few-shot example taught meta-label semantics

gemma4:e4b H6 (baseline) — CORRECT

seizure_types: ["seizure free"]
← Already well-calibrated; few-shot would harm
Why this matters: The H6fs few-shot harness adds three examples targeting this exact failure mode. It improves qwen3.5 by +6.1pp seizure F1 but regresses gemma4 by −3.2pp. Capability-appropriate prompt design is required — there is no universal harness.

BUG: verifier output flat names only → med_full_f1 ≈ 0.018
FIX: verifier preserves dose/unit/frequency → med_full_f1 ≈ 0.60
Why this matters: The original scorer used all-or-nothing tuple matching, which made medication_full_f1 appear catastrophically low (0.372–0.496). After per-component scoring and normalization ("bd" → "twice daily"), the true test performance is 0.724–0.769 — within reach of the Fang et al. ≥0.80 target.

Letter Excerpt (Gan synthetic)

"He reports having 3 seizures over 2 days last month, but otherwise has been seizure free. Prior to that cluster, his last seizure was 4 months ago."

Gold (Pragmatic)

"infrequent"

Gold (Purist)

"1 per 3 month" (normalized from cluster + interval)

GPT-5.5 + Direct label — ERROR

Prediction: "frequent"
← Attends to "3 seizures" without normalizing for cluster context

GPT-5.5 + Retrieval highlight — CORRECT

Prediction: "infrequent"
← Retrieved span highlights "seizure free" and "last seizure was 4 months ago", priming correct normalization
Why this matters: Frequency normalization requires clinical judgment about clusters, intervals, and "otherwise seizure free" qualifiers. Retrieval highlighting directs the model's attention to the disambiguating context, improving exact accuracy from 0.720 to 0.820.
